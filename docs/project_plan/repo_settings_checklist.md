# Repository Settings Checklist

Use this checklist for manual GitHub repo hardening and collaboration hygiene.

## Branch and Merge
- [ ] Protect `main` branch
- [ ] Require PR for merge (if team agrees)
- [ ] Require linear history or squash merge preference
- [ ] Disable force pushes to `main`

## Reviews
- [ ] Require at least one reviewer for non-trivial changes
- [ ] Require conversation resolution before merge (optional)

## Repo Features

- [ ] Discussions enabled (optional)
- [ ] Wiki enabled (optional)

- [ ] Confirm PR template is active
